## ğŸš¢ Deployment

### Production â€” Self-Hosted Kubernetes (GitOps)

The live site at **[el-jefe.me](https://el-jefe.me)** runs on a personal K3s cluster with a full GitOps pipeline:

```
Push to master â†’ GitHub Actions CI â†’ Docker Build & Push â†’ repository_dispatch
  â†’ devops-portfolio-manager updates Helm values â†’ ArgoCD auto-syncs â†’ K8s rollout
```

**Infrastructure stack:**
- **Cluster**: K3s (lightweight Kubernetes)
- **Container**: Multi-arch Docker image (linux/amd64, linux/arm64) served by nginx
- **GitOps**: ArgoCD with automated sync, prune, self-heal, and retry with backoff
- **Helm**: `portfolio-common` shared library chart for standardized deployment templates
- **Ingress**: Traefik with automatic TLS via cert-manager (Let's Encrypt DNS challenge)
- **CDN**: Cloudflare for edge caching and DDoS protection
- **Secrets**: Doppler for CI/CD secret management
- **Image tags**: `YYYYMMDD-HHMMSS-shortsha` format for traceable deployments
- **Contact form**: GraphQL API backend (`GATSBY_CONTACT_FORM_MODE=graphql`)

### Staging â€” Netlify

Netlify is retained as a staging and preview environment:
- Auto-deploys on pushes to the main branch
- Build command: `npm run build:netlify`
- Publish directory: `public/`
- Form handling: Netlify Forms integration (`GATSBY_CONTACT_FORM_MODE=netlify`)
- Headers: CSP and security headers via `_headers` file
- Deploy previews for pull requests

#### Netlify CLI
```bash
npm install -g netlify-cli
npm run build
netlify deploy --prod --dir=public
```
